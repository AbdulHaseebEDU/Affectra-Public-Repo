About Affectra

What is Affectra?

Affectra is an open-source PII (Personally Identifiable Information) exposure self-check platform. It allows individuals to discover where their personal data appears across the public internet — including data breach databases, paste sites, code repositories, social platforms, public directories, and archived web pages. It was designed and built as a Final Year Project at the University of Hertfordshire by Abdul Haseeb Amir (BSc Computer Science, 2026).

What is PII?

Personally Identifiable Information (PII) is any data that can be used — alone or in combination — to identify a specific person. Common examples include:

  • Email addresses — unique identifiers used to register on almost every online service
  • Full names — can be correlated across platforms to build a profile
  • Phone numbers — often exposed in data broker listings and leaked databases
  • Usernames — link activity across forums, social media, and code repositories
  • Passwords (hashed or plain) — the most critical exposure: one leak can compromise many accounts
  • Location data, employer, date of birth — common in data broker and public directory listings

When PII is exposed without consent — through data breaches, careless posting, or commercial data brokers — it can lead to identity theft, phishing attacks, account takeovers, and stalking. The GDPR (General Data Protection Regulation) and CCPA (California Consumer Privacy Act) give individuals the legal right to request removal of their data.

How does Affectra work?

Affectra runs a multi-stage detection pipeline against 12 public intelligence sources. A scan works in four steps:

  1. Query normalisation — your input (email, name, phone, usernames) is cleaned, validated, and standardised before any external call is made.
  2. External API layer — specialised adapters query breach databases (XposedOrNot, HIBP), paste indexers (Psbdmp), social platforms (Gravatar, Holehe, Stack Exchange), code hosts (GitHub), and historical web archives (Wayback Machine, URLScan, crt.sh). Search engine dorks (DuckDuckGo) and an ethical web scraper cover long-tail exposures.
  3. Internal API pipeline — raw findings are normalised, deduplicated, classified into exposure categories (breach, paste, data broker, forum mention, etc.), scored for confidence and risk, and enriched with actionable mitigation steps.
  4. AI analysis — findings are optionally sent to Google Gemini, which generates plain-English threat commentary and finding-specific remediation actions tailored to exactly what was found.

Scan modes

  • API Only — fastest; queries intelligence APIs only. Best for a quick breach check.
  • Hybrid — APIs + search engine + ethical scraping of returned URLs. The recommended default.
  • Deep Scan — same as Hybrid but with higher per-source limits for broader coverage.
  • Extended Exploration — recursive mode: newly discovered identifiers (linked emails, usernames) are automatically fed back into the pipeline for follow-up scans.
  • Compare All Modes — runs all four modes and shows side-by-side coverage metrics.

Privacy & ethics

Affectra is designed to be used only on data that belongs to you. Every external source queried is a publicly accessible internet service. The tool respects robots.txt, applies per-host rate limiting, and never stores your input beyond the duration of a scan session. Mitigation steps include GDPR Article 17 deletion email templates where applicable, giving you a direct path to request removal of your data from each source.
